🌐 1. La Red Global y la Autonomía

Internet no es una red centralizada, sino una "red de redes". Aquí nace el concepto de Sistema Autónomo (AS). Un AS es una gigantesca infraestructura (como Antel o una universidad) con una política de enrutamiento propia. Poseen autonomía para manejar su tráfico interno y utilizan el protocolo BGP para conectarse con otros AS, tejiendo la red global.

🗺️ 2. Topologías Base

La topología define el mapa físico o lógico de conexión. La Estrella depende de un nodo central (alta vulnerabilidad si falla el centro), mientras que la Malla (Mesh) ofrece múltiples caminos redundantes, siendo la base robusta de Internet.

⚙️ 3. Hardware de Red

Para construir las topologías, requerimos dispositivos que operan en diferentes niveles de inteligencia.

🖧

Switch (Capa 2)

Cerebro de la LAN. Aprende Direcciones MAC y envía datos solo al puerto correcto, evitando saturación.

🌐

Router (Capa 3)

Dispositivo de frontera. Lee direcciones IP, enruta paquetes entre redes distintas (WAN) y asigna IPs (DHCP).

📡

Access Point

Puente inalámbrico. Transforma la red cableada en ondas de radio (WiFi) para dispositivos móviles.

📦 4. Segmentos: TCP vs UDP

En la Capa 4 (Transporte) del Modelo OSI, los archivos grandes se dividen en Segmentos y se etiquetan con Puertos. Existen dos metodologías principales para enviar estos segmentos:

  • ✓ TCP (Seguro): Enumera segmentos, asegura entrega total y retransmite si hay pérdidas. Ideal para Webs y PDFs.
  • ⚡ UDP (Rápido): Envío continuo sin confirmación. No retransmite. Ideal para VoIP y Juegos Online donde la velocidad es crítica.

🛠️ 5. El Arsenal de Diagnóstico CLI

> ping -t 192.168.1.1

Detecta cortes intermitentes y Packet Loss (Pérdida de paquetes). Un cable dañado o interferencia WiFi severa generará tiempos de espera agotados.

> tracert 8.8.8.8

Rastrea los saltos de enrutamiento. Identifica cuellos de botella observando picos de latencia en nodos intermediarios (ej. el salto del ISP local).

ipconfig /flushdns

Comando salvavidas en Windows. Borra la memoria caché de direcciones web de tu PC, obligando al sistema a resolver las rutas de red desde cero si un servidor cambió de IP.

tcpdump -i eth0 icmp

Herramienta de análisis profundo (Sniffing) en Linux. Captura paquetes en tiempo real en la tarjeta de red para comprobar empíricamente si la información está saliendo del equipo.
